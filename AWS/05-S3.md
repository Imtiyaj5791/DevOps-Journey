# 1. What is Amazon S3?

Amazon S3 stands for **Simple Storage Service**.

S3 is an **object storage service** provided by AWS. It is used to store data as objects inside buckets.

Common use cases:

- Backups
- Application logs
- Images and videos
- Documents
- Static files
- Archive data
- Large amounts of object data

### Interview Answer

> Amazon S3 is an object storage service provided by AWS. It stores data as objects inside buckets. We commonly use S3 for backups, logs, images, documents, static files and archival data. S3 provides high durability and scalable storage.

---

# 2. S3 Bucket and Object

## Bucket

A bucket is a **container used to store objects in Amazon S3**.

Example:

```text
Bucket: imtiyaj-s3-practice-2026
```

Important:

- Bucket names must be globally unique.
- Objects are stored inside buckets.
- Buckets are created in an AWS Region.

## Object

An object is the actual data stored inside an S3 bucket.

Examples:

```text
Docker.docx
application.log
backup.tar.gz
report.pdf
```

An object contains:

- Data
- Object Key
- Metadata

### Interview Answer

> An S3 bucket is a container where we store objects. An object is the actual data such as images, documents, logs or backup files stored inside the bucket.

---

# 3. Object Key

An **Object Key** uniquely identifies an object inside a bucket.

Example:

```text
Docker.docx
```

If an object appears inside a folder-like structure:

```text
documents/aws/Docker.docx
```

The complete object key is:

```text
documents/aws/Docker.docx
```

S3 does not use traditional filesystem directories. It uses **key prefixes** to provide a folder-like structure.

---

# 4. S3 URI

Example:

```text
s3://imtiyaj-s3-practice-2026/Docker.docx
```

Breakdown:

```text
Bucket     = imtiyaj-s3-practice-2026
Object Key = Docker.docx
```

S3 URIs are commonly used with AWS CLI and AWS services.

Example:

```bash
aws s3 cp Docker.docx s3://imtiyaj-s3-practice-2026/
```

---

# 5. Object URL

S3 objects can have an Object URL.

Having an Object URL does **not** mean that the object is publicly accessible.

If public access is blocked and the user does not have permission, accessing the Object URL can return:

```text
Access Denied
```

# 6. S3 Storage Classes

S3 provides different storage classes based on:

- Access frequency
- Retrieval requirement
- Availability requirement
- Storage cost
- Archival requirement

---

## 6.1 S3 Standard

Used for **frequently accessed data**.

Features:

- Multi-AZ
- High availability
- Millisecond access
- 99.999999999% (11 9s) durability
- Suitable for frequently accessed objects

Examples:

- Active application files
- Frequently accessed documents
- Images
- Website content

### Interview Answer

> S3 Standard is used for frequently accessed data. It provides high availability, Multi-AZ storage and 11 9s durability.

---

## 6.2 S3 Standard-IA

IA stands for **Infrequent Access**.

Used when data is accessed infrequently but must be available immediately when required.

Features:

- Multi-AZ
- Millisecond retrieval
- Lower storage cost than S3 Standard
- Retrieval charges apply
- Minimum storage duration: 30 days

Examples:

- Backups
- Disaster recovery files
- Infrequently accessed business data

### Interview Answer

> S3 Standard-IA is used for data that is accessed infrequently but needs immediate retrieval when required. Storage cost is lower than S3 Standard, but retrieval charges apply.

---

## 6.3 S3 One Zone-IA

Used for infrequently accessed data stored in a **single Availability Zone**.

Features:

- Single AZ
- Immediate retrieval
- Lower cost than Standard-IA
- Retrieval charges apply
- Minimum storage duration: 30 days
- Suitable for re-creatable data

Because the data is stored in one AZ, it should not normally be used as the only copy of critical data.

### Interview Answer

> S3 One Zone-IA is used for infrequently accessed and re-creatable data. It stores data in a single Availability Zone and costs less than Standard-IA.

---

## 6.4 S3 Intelligent-Tiering

Used when the access pattern of data is **unknown or changes over time**.

S3 monitors access patterns and automatically moves objects between appropriate access tiers to optimize storage cost.

Features:

- Automatic cost optimization
- Useful when access pattern is unpredictable
- Monitoring and automation charges apply

### Interview Answer

> S3 Intelligent-Tiering is useful when we don't know how frequently data will be accessed. AWS automatically moves objects between access tiers based on usage to optimize storage cost.

---

# 7. S3 Glacier Storage Classes

Glacier storage classes are mainly used for **archive data**.

---

## 7.1 Glacier Instant Retrieval

Used for archive data that is rarely accessed but needs **millisecond retrieval** when required.

```text
Archive Data
+
Rarely Accessed
+
Millisecond Retrieval
```

---

## 7.2 Glacier Flexible Retrieval

Used for archive data where immediate retrieval is not required.

Retrieval can range from **minutes to hours**, depending on the retrieval option.

```text
Archive Data
+
Rarely Accessed
+
Minutes/Hours Retrieval
```

---

## 7.3 Glacier Deep Archive

Used for **long-term archival data** that is very rarely accessed.

Examples:

- Long-term compliance records
- Old financial records
- Historical backups
- Long-term company records

Retrieval normally takes hours.

```text
Long-Term Archive
+
Very Rare Access
+
Hours Retrieval
+
Very Low Storage Cost
```

---

# 8. Storage Class Quick Comparison

| Storage Class | Access Pattern | AZ | Retrieval |
|---|---|---|---|
| S3 Standard | Frequent | Multi-AZ | Milliseconds |
| Standard-IA | Infrequent | Multi-AZ | Milliseconds |
| One Zone-IA | Infrequent | Single AZ | Milliseconds |
| Intelligent-Tiering | Unknown/Changing | Multi-AZ | Based on tier |
| Glacier Instant | Rare Archive | Multi-AZ | Milliseconds |
| Glacier Flexible | Archive | Multi-AZ | Minutes to Hours |
| Glacier Deep Archive | Long-term Archive | Multi-AZ | Hours |

# 9. S3 Versioning

S3 Versioning maintains **multiple versions of the same object**.

Example:

```text
Docker.docx → Version 1
Docker.docx → Version 2
Docker.docx → Version 3
```

Benefits:

- Protection against accidental overwrite
- Protection against accidental deletion
- Previous versions can be recovered
- Each version gets a Version ID

### Interview Answer

> S3 Versioning keeps multiple versions of the same object. If an object is accidentally overwritten, we can recover the previous version. If an object is deleted, previous versions remain available for recovery.

---

# 10. Delete Marker

When Versioning is enabled and an object is normally deleted, S3 creates a **Delete Marker**.

Example:

```text
Docker.docx → Delete Marker
Docker.docx → Previous Version
Docker.docx → Older Version
```

The previous versions still exist.

Removing the Delete Marker makes the previous latest version visible again.

# 12. S3 Lifecycle Policy

S3 Lifecycle Policy automatically manages objects based on their age.

Two important lifecycle actions:

## Transition

Automatically move objects between storage classes.

Example:

```text
S3 Standard
     ↓
Standard-IA
     ↓
Glacier Flexible
     ↓
Glacier Deep Archive
```

## Expiration

Automatically expire/delete objects after a configured period.

Example:

```text
Object
  ↓
365 Days
  ↓
Expire/Delete
```

---

# 13. Lifecycle Policy Example

Suppose application logs are uploaded to S3.

Requirement:

```text
Day 1 - 30
S3 Standard

Day 31 - 90
Standard-IA

Day 91 - 180
Glacier Flexible Retrieval

Day 181+
Glacier Deep Archive
```

Lifecycle configuration:

```text
30 Days  → Standard-IA
90 Days  → Glacier Flexible
180 Days → Glacier Deep Archive
```

Important:

The number of days refers to the **age of the object from its creation/upload**, not the number of days spent in the previous storage class.

---

# 14. Lifecycle with Versioning

Versioning can create multiple old versions of objects.

These old versions consume storage.

Lifecycle rules can manage:

- Current object versions
- Noncurrent object versions
- Delete markers
- Incomplete multipart uploads

For example:

```text
Old Version
    ↓
Transition to cheaper storage
    ↓
Expire after retention period
```

This helps reduce unnecessary S3 storage cost.

# 16. Day 01 Important Interview Questions

### Q1. What is Amazon S3?

Amazon S3 is an object storage service used to store data as objects inside buckets.

### Q2. What is the difference between a bucket and an object?

A bucket is a container, while an object is the actual data stored inside the bucket.

### Q3. What is an Object Key?

An Object Key uniquely identifies an object inside an S3 bucket.

### Q4. Which storage class is suitable for frequently accessed data?

S3 Standard.

### Q5. Which class is suitable for infrequently accessed data requiring immediate retrieval?

S3 Standard-IA.

### Q6. Standard-IA vs One Zone-IA?

Standard-IA stores data across multiple AZs, while One Zone-IA stores data in a single AZ and costs less.

### Q7. When would you use Intelligent-Tiering?

When the access pattern is unknown or changes over time.

### Q8. What is S3 Versioning?

Versioning maintains multiple versions of the same object and helps recover from accidental overwrite or deletion.

### Q9. What happens when we delete an object from a version-enabled bucket?

S3 normally creates a Delete Marker while previous versions remain available.

### Q10. What is an S3 Lifecycle Policy?

It automatically transitions objects between storage classes or expires them based on configured rules.

### Q11. Why do we use Lifecycle Policies?

To automate data management and optimize S3 storage costs.
